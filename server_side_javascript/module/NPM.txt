Node Package Manager의 약자로써, 다른 사람이 제작한 module를 가져다가 사용하는 방법에 대해 관리하는 방법을 배운다.
언어에 존재하는 기본적인 기능을 제공하는 부품이 module이다.

타인의 모듈을 사용하는 방법 => NPM -> Node계의 앱 스토어이다.
NPM의 기능은 설치, 삭제, 업그레이드, 의존성 관리(다른 사람의 모듈들에 우리가 의존한다.)등을 해준다.

module는 독립적으로 사용이 되거나, 어딘가의 부품으로 사용이 되는 걸로 나뉘게 된다.
2가지를 모두 합쳐서 package라고 한다.

module중에 uglify-js라는 module이 있는데, 홈페이지에서 검색을 해서 들어가면,
npm install uglify-js -g 로 install이 가능하다. -g는 global이다. 컴퓨터 전역에서 사용이 가능하다.(독립적인 소프트웨어)
-g를 붙이지 않으면, 현재 해당하는 곳에서만 부품으로 사용하겠다는 의미이다.


cmd에 npm install uglify-js -g를 입력을 하게 되면, 설치가 된다. 이제 어느 디렉토리에서든 사용이 가능하게 된다.

예시로 pretty.js파일을 보자.
----------------------------------------------------------------------------------------------------------------

원하는 js에 내가 원하는 모듈을 사용할 수 있어야 하기에 package를 직접적으로 설정을 해줄 수 있다.
cmd에서 npm init을 입력한다. 원하는 입력 값은 ()안에 자동으로 채워지기에 enter를 치면 되는데, description의 경우 직접적으로 입력한다.
entry point: 부분이 모듈을 내가 원하는 js파일에 적용을 시킬 수 있는 부분인데, 나중에 바꾸는 것도 가능하다.(package.json파일 들어가서 바꾸면 된다.)
--> package.json이라는 파일이 생성이 된다.
